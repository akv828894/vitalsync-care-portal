# Interview Prep

## 1. Explain JWT

Short answer:

`JWT` is a token generated by the backend after successful login. The frontend sends this token in future requests so the server can verify the user's identity without asking for the password again.

## 2. Explain bcrypt

Short answer:

`bcryptjs` hashes the password before saving it in MongoDB. That means even if the database is leaked, the real password is not directly visible.

## 3. Explain Middleware

Short answer:

Middleware is a function that runs before the final route handler. In auth systems, middleware checks whether the incoming request has a valid JWT before protected data is returned.

## 4. Explain localStorage

Short answer:

In this internship MVP, `localStorage` is used because it is easy for beginners and simple to demonstrate. In production, `HttpOnly` cookies are usually safer because JavaScript cannot read them directly.

## 5. Explain Protected Routes

Short answer:

A protected route is a route that only logged-in users can access. On the frontend we redirect unauthorized users to `/login`, and on the backend we block access using JWT middleware.

## 7. Strong Answers You Should Practice

### What is the difference between authentication and authorization?

Authentication means proving who the user is, usually through login. Authorization means deciding what that logged-in user is allowed to access.

### Why do we return a token after login?

Because the frontend needs a secure way to prove the user is already logged in for later requests to protected routes.

### Why is `/api/patients/overview` protected?

Because patient-related data should only be accessed by an authenticated user. Otherwise anyone could access sensitive information directly.

### Why use a model file?

Because the model defines the database structure, validation rules, and schema methods like password comparison.
